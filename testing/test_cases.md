 Тест-кейсы функционального тестирования

Модуль: Управление клиентами

: Добавление нового клиента
Приоритет: Высокий
Предусловия: Приложение запущено, открыта главная форма

| Шаг | Действие | Ожидаемый результат |

| 1 | Нажать "Добавить клиента" | Открывается форма ввода данных |
| 2 | Заполнить поля: Иван Иванов, +79991234567 | Данные принимаются |
| 3 | Нажать "Сохранить" | Клиент добавляется в список |

 : Создание заявки на ремонт
Приоритет: Высокий
Предусловия: Есть минимум один клиент в системе

| Шаг | Действие | Ожидаемый результат |

| 1 | Выбрать клиента из списка | Клиент выделяется |
| 2 | Нажать "Новая заявка" | Открывается форма заявки |
| 3 | Заполнить данные автомобиля | Поля принимают данные |
| 4 | Выбрать тип ремонта | Тип сохраняется |
| 5 | Нажать "Создать заявку" | Заявка появляется в списке |

 Модуль: Управление статусами

: Изменение статуса заявки
Приоритет: Средний
Предусловия: Есть заявка со статусом "Новая"

| Шаг | Действие | Ожидаемый результат |

| 1 | Выбрать заявку в списке | Заявка выделяется |
| 2 | Нажать "Изменить статус" | Открывается меню выбора |
| 3 | Выбрать "В работе" | Статус обновляется в интерфейсе |
| 4 | Обновить список | Новый статус отображается |
